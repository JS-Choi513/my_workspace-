---

구현 계획서 검토에서 워크스페이스 하네스 문서 체계의 공통 약점을 발견하였으므로 향후 검토 시 집중할 부분.

**Why:** 체계적인 문서화는 되어있으나 실행 가능성과 역할 경계 명확화에서 반복적 문제 패턴 관찰

**How to apply:** 향후 구현 계획서 검토 시 다음 항목들을 우선적으로 점검
- 에이전트 간 역할 겹침 (특히 에러 처리 vs 판단 에스컬레이션)
- 문서 간 우선순위 충돌 (글로벌 vs 프로젝트별 규칙)
- 추상적 지시사항 (구체적 실행 정보 누락)
- 파일 참조 무결성 (존재하지 않는 파일 참조)
- 용어 일관성 (대소문자, 개념 정의 통일)
